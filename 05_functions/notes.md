# 第5章 学習メモ

## 学習期間
2026/03/29 〜 2026/04/12

## 理解できたこと
- def で関数を定義し、何度でも呼び出せること。関数内で他の関数を呼べるので処理を階層化しやすい。
- 引数は位置・キーワード・デフォルト・可変長で柔軟に渡せる。キーワード引数は順序を気にせず読みやすい。
- return で値を返し、早期 return（ガード節）や複数値（タプル）で関数挙動を明確にできる。
- スコープ（ローカル/グローバル）の仕組みを理解し、global は最低限にして引数/戻り値で状態を渡すほうが扱いやすい。
- lambda式(無名関数) と高階関数（sorted, map, filter, 関数を返す関数）を用いると簡潔に記述できる。
- sorted や map, filter にラムダを渡して、辞書リストや数値リストを柔軟に操作できる。
- クロージャー（make_multiplier, make_tax_calculator）で状態を閉じ込めた関数を生成できる。

## 詰まったこと・疑問
- global の使いどころと副作用がまだ少し気になる。引数で受け渡す習慣をもっと徹底したい。
- 可変長引数とキーワード引数を混ぜると順序がわかりづらくなるので、使い分けのルールを自分なりに整理する必要がある。

## 気づき・応用アイデア
- default 引数 + キーワード引数を組み合わせてオプションを整え、外部 API ラッパーのような柔軟な呼び出し方ができそう。
- 関数を返す設計は設定値（税率、倍率など）を閉じ込めて汎用的な処理を渡せるので再利用性が高い。
- map/filter で簡潔にリスト加工するとイテレーション回数が減るので、読みやすさも性能も改善できそう。
